Man in the middle

MITM is quite popular attack with regards to network devices in general, the attack intercepts communication between two nodes and allows the attacker to take the role of a proxy .In IoT, the attacker usually performs MITM attacks between an IoT device and the application with which it interfaces. IoT devices are vulnerable to MITM as they lack the standard implementations to fight the attacks. MITM attacks can be conducted in several ways namely Address Resolution Protocol (ARP) cache poisoning, DNS spoofing, Session hijacking including side-jacking, evil twin, sniffing, SSL Hijacking. As per IoT is concern there are name two modes of attack : cloud polling and direct connection. Many IoT devices do not verify the authenticity or the trust level of certificates which makes easier for attacker to do cloud polling , the smart device which has data being shared with cloud, usually look for firmware updates. Attackers can redirect network traffic using Address Resolution Protocol (ARP) poisoning or by altering Domain Name System (DNS) settings or intercept HTTPS traffic by using self-signed certificates or tools such as (Secure Sockets Layer) SSL strip. In the case of direct connections, devices communicate with a hub in the same network. By doing this, mobile apps can locate new devices by probing every IP address on the local network for a specific port. An attacker can do the same thing to discover devices on the network. Many IoT devices uses smaller, cheaper, and more power-efficient called Bluetooth Low Energy (BLE). However, BLE is vulnerable to MITM attacks. AES-CCM encryption which BLE uses, Key exchanged is often insecure. Once when an attacker has access to some of the devices on an IoT network via an MITM attack, they could carry out False Data Injection (FDI) attack. An attacker alters measurements from IoT sensors by a small amount so as to avoid suspicion and then outputs the faulty data. FDI attacks are commonly exploited against sensors that send data to an algorithm that tries to make a prediction of based on the data it has received, or uses the data to draw a conclusion. These algorithms, sometimes referred to as predictive maintenance systems, are commonly used to monitor the condition of a mechanical and predict when it needs service or adjustment. These predictive maintenance algorithms and the like will also be a fundamental feature of the IDE attacks that could be catastrophic. .

DDoS

Many attackers use botnets and launch Distributed Denial of Service (DDoS) attacks. A denial of service (DoS) attack is characterized by an orchestrated effort to prevent legitimate use of a service. DDoS attacks aim to overwhelm the infrastructure of the target service and disrupt normal data flow. DDoS attacks go through few phases : Recruitment : In which the attacker scans for vulnerable devices to be used in the DDoS attack against the target. Exploitation and infection: In which the vulnerable devices are injected with malicious code is injected to disrupt the data communication, in which the attacker assesses the device . The attacker injected with malicious code to a vulnerable machine commands the infected machines to send malicious packets to the target. One of the most famous malware, the Mirai worm, has been used to perpetrate and is designed to infect and control IoT devices such as CCTV cameras, and smart home routers. The infected devices become part of a large-scale botnet and can perpetrate several types of DDoS attacks.
